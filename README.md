A practical demonstration and testbed designed and built autonomously by AI agents under strict software engineering guidelines (**Anti-Vibecoding**). 

This project serves as a proof of concept to validate the evolution of AI personas (Architect, Backend, and Frontend) using advanced, enterprise-grade design patterns, including real database concurrency controls, multi-tenant data isolation, and a highly polished multi-state UI.

## 🛠️ Application & Architecture

The application simulates a **Simplified Financial Dashboard** split into two main layers:

### 1. Backend (`/backend`) - FastAPI + SQLite (SQLAlchemy)
Implemented by `@persona-backend` with a focus on safety, security, and data consistency:
* **Double Spending Prevention (Race Conditions)**: Balance transfers are executed within explicit database transactions using write locks (`with_for_update`) during account queries.
* **Rigorous Isolation (Anti-IDOR)**: Every database query is strictly scoped by the `tenant_id` extracted from the request header (simulating a JWT token). A user from one tenant cannot read or write to another tenant's account.
* **Border Validation**: Powered by Pydantic to enforce strict types, decimal limits (`max_digits=12`, `decimal_places=2`), and sanitized strings.

### 2. Frontend (`/frontend`) - React (SPA via CDN) + Tailwind CSS
Implemented by `@persona-frontend` with high visual fidelity and robust state handling:
* **Premium UI/UX**: Designed with a sleek, dark mode interface (Deep Dark), Google Fonts' Outfit typography, and clean glassmorphism styling.
* **4 Logical States Handling**:
  * **Success**: Renders real balances and transaction logs.
  * **Loading**: Uses customized, animated skeleton UI elements to prevent layout shifts.
  * **Error**: Displays human-readable error messages alongside a functional Retry CTA.
  * **Empty**: Clean visual feedback if the account has no transactions.
* **Browser Security**: Native string escaping to protect against visual Cross-Site Scripting (XSS) in transactions.
* **Interactive Testing Panel**: A side panel allows switching between tenants and simulating edge cases (such as insufficient balance or unauthorized tenant switching) to test the app's guardrails instantly.

## 🚀 How to Run Locally

### Step 1: Run the Backend

Make sure you have Python 3.10 or higher installed.

1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Create and activate a virtual environment (optional but recommended):
   ```bash
   python -m venv .venv
   # On Windows (PowerShell):
   .venv\Scripts\Activate.ps1
   # On Linux/macOS:
   source .venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Start the server:
   ```bash
   uvicorn app:app --reload --port 8000
   ```
   The backend API will be available at `http://localhost:8000`.

### Step 2: Run the Frontend

Since the frontend is a Single Page Application using React and Tailwind CSS directly via CDN, it does not require Node.js or npm packages.

1. Simply open the `frontend/index.html` file in your preferred web browser (e.g., by double-clicking it or using an extension like Live Server in VS Code).
2. Ensure the backend server is running at `http://localhost:8000`.
